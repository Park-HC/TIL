# 데이터 베이스

## 특징

| 특징                     | 설명                                                         |
| ------------------------ | ------------------------------------------------------------ |
| 통합 데이터(Integrated)  | 데이터베이스에 같은 내용의 데이터가 중복되어 있지 않음       |
| 저장 데이터(Stored)      | 컴퓨터가 접근할 수 있는 저장 매체에 저장                     |
| 공용 데이터(Shared)      | 여러 사용자들에게 서로 다른 목적으로 DB 데이터를  공동 이용  |
| 변화되는 데이터(Changed) | 추가, 삭제, 갱신으로 항상 변화하면서도 현재의 정확한 데이터를 유지해야 함 |



## DBMS

| 용어  | 설명                                                         |
| ----- | ------------------------------------------------------------ |
| DBMS  | 사용자와 데이터베이스 사이에서 사용자의 요구에 따라 정보를 처리하는 소프트 웨어 |
| RDBMS | 관계형 데이터베이스 관리 시스템<br />정형화된 테이블로 구성된 데이터 항목들의 집합체<br />MySAL, Oracle Database 등<br />SQL로 조작 |
| ODBMS | 객체 지향 데이터베이스 관리 시스템<br />객체들을 생성하여 계층에서 체계적으로 정리<br />다시 계층들을 하위 계층이 상위 계층으로부터 속성과 방법들을 물려받는(상속) DBMS<br />**복잡한 데이터 구조**를 표현 및 관리하는 DBMS |



## 데이터베이스 설계 절차

| 순서 | 명칭           | 설명                                                         |
| ---- | -------------- | ------------------------------------------------------------ |
| 1    | 요구 조건 분석 | 데이터베이스 사용자, 사용목적, 범위, 제약조건 등을 정리<br />명세서 작성 |
| 2    | 개념적 설계    | E-R 모델, 정보를 추상적 개념으로 표현하는 과정<br />DBMS 독립적 E-R 다이어그램 작성 |
| 3    | 논리적 설계    | 자료를 컴퓨터가 이해할 수 있도록 특정 DBMS의 논리적 자료 구조로 변환 |
| 4    | 물리적 설계    | 논리적 구조로 표현된 데이터를 물리적 구조의 데이터로 변환하는 과정 |

> **설계된 데이터를 분석**한 결과, **요구 조건**이 **개념** 없으면 **논리**(**물리**)로 승부를 봐야한다!



## NoSQL

- 관계형 데이터베이스보다 덜 제한적인 일관성 모델 이용
- 데이터의 저장 및 검색을 위한 매커니즘 제공
- 디자인 단순환
- 수평적 확장성
- 세세한 통제
- RDBMS 특성 + 다른 특성들

### 저장 방식 도구

| 도구         | 설명                                                         |
| ------------ | ------------------------------------------------------------ |
| MongoDB      | 데이터 교환 시 비산(Binary Json) 문서 형태로 저장<br />여러 서버에 분산 저장 및 확장 용이<br />방대한 데이터 처리가 빠름<br />C++ |
| Apache HBase | 하둡 플랫폼을 위한 공개 비관계형 분산 데이터 베이스<br />구글의 빅테이블을 본보기로 삼음<br />Java |
| Redis        | Remote Dictionary Server의 약자<br />"키-값" 구조의 비정형 데이터 저장, 관리<br />오픈 소스 기반의 비관계형 DBMS |



# 기업 내부 데이터베이스 솔루션

## 시대별 분류

### 1980년대

| 솔루션 | 설명                                                         |
| ------ | ------------------------------------------------------------ |
| OLTP   | Online Transaction Processing<br />온라인 거래 처리<br />주 컴퓨터와 통신회석으로 접속되어 있는 복수의 사용자 단말에 발생한 트랜잭션을<br />주 컴퓨터에서 처리해 그 결과를 사용자에 되돌려 보내 주는 처리형태 |
| OLAP   | Online Analytical Processing<br />온라인 분석 처리<br />다차원으로 이루어진 데이터로부터 통계적인 요약 정보를 제공할 수 있는 기술<br />다차원 데이터를 대화식으로 분석하기 위한 SW |



### 2000년대

| 솔루션 | 설명                                                         |
| ------ | ------------------------------------------------------------ |
| CRM    | Customer Relationship Management<br />고객별 구매 이력 데이터 베이스 분석<br />고객에 대한 이해를 돕고 이를 바탕으로 마케팅 전략 수립<br />보다 높은 이익을 창출 |
| SCM    | Supply Chain Management<br />제조, 물류, 유통업체 등 유통공급망에 참여하는 모든 업체들이 협력을 바탕으로 정보 기술 활용<br />재고를 최적화하기 위한 솔루션<br />기업이 외부 공급업체 또는 제휴업체와 통합된 정보시스템으로 연계<br />시간과 비용 최적화<br />자재구매 데이터, 생산, 재고 데이터, 유통/판매 데이터, 고객 데이터로 구성 |



## 분야별 분류

### 제조업

| 솔루션         | 설명                                                         |
| -------------- | ------------------------------------------------------------ |
| Data Warehouse | 기업 내 의사결정 지원 애플리케이션을 위한 정보를 제공하는 하나의 통합된 데이터 저장 공간<br />**ETL(추출, 변환, 적재)**: 주기적으로 내부 및 외부 데이터베이스로부터 정보를 추출하고 정해진 규약에 따라 정보를 변환 후 정보 적재<br />데이터들은 시간적 흐름에 따라 변화하는 값 유지 |
| Data Mart      | 전사적 Data Warehouse로부터 특정 주제, 부서 중심으로 구축된 **소규모 단일 주제 DW**<br />재무, 생산, 운영과 같이 특정 조직의 특정 업무 분야에 초점을 둠 |
| ERP            | Enterprise Resource Planning<br />제조업을 포함한 다양한 biz 분야에서 생산, 구매, 재고, 주문, 거래, 고객 서비스 제공 등<br />주요 프로세스 관리를 돕는 여러 모듈로 구성된 통합 어플리케이션 SW 패키지 |
| BI             | Business Intelligence<br />기업의 Data Warehouse에 저장된 데이터에 접근해 경영의사결정에 필요한 정보를 획득하고 이를 경영에 활용하는 것<br />데이터를 통합, 분석하여 기업 활동에 연관된 의사결정을 돕는 프로세스<br />**가트너**: 사용자가 필요로 하는 정보를 정확한 시간에 제공할 수 있는 환경<br />**하나의 특정 비즈니스 질문**에 답변하도록 설계 |
| BA             | Business Analytice<br />경영 의사결정을 위해 통계적이고 수학적인 분석에 초점을 둔 기법<br />성과에 대한 이해와 비즈니스 통찰력에 초점을 둔 분석 방법<br />사전에 예측하고 최적화하기 위한 것으로 **BI보다 진보된 형태** |
| ad hoc report  | BI와 빅데이터 분석의 차이점을 표현한 키워드<br />일회용으로 작성된 임시 보고서<br />Optimization, forecast, insight: 빅데이터 분석 관련 키워드 |



#### Dtat Warehouse 4대 특성

- 데이터 통합: 데이터들은 전사적 차원에서 일관된 형식으로 정의됨
- 데이터의 시계열성: 시간의 흐름에 따라 변화하는 값 저장
- 데이터 주제 지향성: 특정 주제에 따라
- 데이터 비소멸성(비휘발성): 데이터들이 분류, 저장 관리 됨



### 금융

#### 블록체인

- 기존 금융회사의 중앙집중형 서버에 거래 기록을 보관하는 방식에서 벗어나 거래에 참여하는 모든 사용자에게 거래 내용을 보내주며 거래 때마다 이를 대조하는 데이터 위조 방지 기술
- EIA, EDW, ERP, e-ERM 등도 있음
